Microservices architecture is a design pattern where an application is composed of many small, independent services communicating over well-defined APIs. Unlike monolithic architectures, where all code is packaged into a single deployable unit, microservices allow individual teams to develop, deploy, and scale services independently. A key principle is decentralised data management; each service should own its own database to prevent tight coupling. Communication is typically handled via synchronous HTTP/REST or asynchronous message brokers like Kafka or RabbitMQ. While microservices offer immense scalability, they introduce complexity in monitoring, distributed tracing, and transaction management.
